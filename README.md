針對 **`basic-04-unit-pwm-basics` (ESP32 PWM 基礎：占空比的魔法)** 單元，這是學員從「恆定輸出」轉向「按需分配能量」的認知轉折。它要求學員具備 **能量調變 (Energy Modulation)** 的概念，學習如何利用高頻開關模擬類比效果，並解決物理世界中如 **「靜摩擦力 (Static Friction)」** 與生物世界中 **「人眼非線性感知 (Visual Non-linearity)」** 的補償問題。

以下是在 **GitHub Classroom** 部署其作業 (Assignments) 的具體建議：

### 1. 範本倉庫 (Template Repo) 配置建議
PWM 基礎作業的核心在於「公式對應」與「補償實作」，範本應包含實驗性的數據記錄模板，建議包含：
*   **📂 `src/main.cpp`**：**核心實作區**。學員需在此實作一個具備「死區補償 (Dead-zone Compensation)」的馬達驅動函式 `smartDrive(int power)`，以及兩種模式的呼吸燈演算法（線性 vs. Gamma 校正）。
*   **📂 `include/params.h`**：參數配置中心。供學員填寫實驗後得出的數據，如 `MOTOR_START_OFFSET` (馬達起步門檻)。
*   **📂 `README.md`**：**實驗量測報告**。提供表格範本讓學員記錄：「占空比 (%)」 vs 「計算出的平均電壓 (V)」。並撰寫「Gamma 校正」前後的視覺差異觀察。
*   **📂 `docs/CALIBRATION.md`**：**死區量測紀錄**。學員需詳細描述如何找到馬達開始克服摩擦力「轉動瞬間」的 PWM 最小值，並以此作為軟體補償的依據。

### 2. 作業任務部署細節

#### 任務 1：「捕捉脈衝」波形觀測實驗 (Pulse Audit Lab)
*   **目標**：透過數據換算建立「平均電壓」的物理直覺，理解 PWM 能量傳遞的本質。
*   **Classroom 部署建議**：
    *   **公式驗證**：要求學員在不同占空比 (0%, 25%, 50%, 75%, 100%) 下計算理應得到的平均電壓值 $V_{avg} = 3.3V \times Duty\%$。
    *   **現象觀察**：讓學員將 PWM 頻率調低至 1Hz，觀察 LED 閃爍與馬達「抽搐」的現象，理解高頻切換的必要性。
    *   **驗證要點**：在 README 中彙整完整數據表，建立對於「寬度 = 能量」的深刻映射。

#### 任務 2：「視覺平衡」Gamma 校正呼吸燈 (Lighting UX Lab)
*   **目標**：認識人眼的非線性感知特性，學習如何用數學公式（Gamma 校正）提升使用者體驗。
*   **Classroom 部署建議**：
    *   **呼吸燈對比**：實作兩組呼吸燈。A 組為「線性增加 (0-255)」，B 組為「平方增加 (i*i/255)」。
    *   **感知分析**：觀察 A 組是否在亮區停頓感過長？而 B 組是否感覺更均勻平滑？
    *   **驗證要點**：提交一份簡報或 Markdown 描述，說明非線性映射在「細膩 UI 燈效設計」中的不可替代性。

#### 任務 3：「啟動突破」克服靜摩擦力 (Motor Power Break & SmartDrive Lab)
*   **目標**：實作專業的韌體補償，解決硬體「起步死區」問題，這是開發「平滑啟動無人車」的核心技術。
*   **Classroom 部署建議**：
    *   **死區尋找**：逐步增加 PWM，直到馬達發出低鳴並「開始克服摩擦力轉動」瞬間，紀錄該數值 (例如為 40)。
    *   **補償封裝**：撰寫 `smartDrive(int speed)`。公式為 `finalPWM = Offset + speed * (255 - Offset) / 255`。
    *   **效果驗證**：確保當下達「速度 1%」指令時，馬達能立即以極低速穩定轉動，而非卡死發燙。
    *   **最終 Commit**：封裝完整的動力補償邏輯並 Push。
